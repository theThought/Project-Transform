Accessibility considerations & other notes

Specifically, wrt the use of <table> markup.

  • Tables can be given an accessible name either by an explicit <caption> or aria-labelledby. ARIA wins if both are present. Note that aria-describedby does not work (in NVDA).
  • Table header cells <th> are announced both with and without the scope attribute.
  • Table header cells are not sufficient to provide an accessible name for form controls, even with the addition of headers attributes.
  • Real <label for=""> must be used to provide an accessible name for form controls.

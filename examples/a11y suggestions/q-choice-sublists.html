Accessibility considerations & other notes

  • The question cannot be a <label>, as it relates to multiple form controls.
    • Instead, it has an id so that it can be programmatically associated with the sublist <fieldset> using aria-describedby.
  • Sublists are grouped inputs (checkboxes/radios), and therefore should use <fieldset><legend>.
  • Additional instructions... these can go below the question, just like "q-single-edit.html".
  • The fieldsets have an aria-describedby attribute to programmatically associate them with a unique id on the instructions. This enables screen readers to announce the instructions in addition to the fieldset legends.
  • If status message is an error:
    • Place error message before question.
    • Shift keyboard focus to the 1st error message on the page so that it's the 1st thing announced to screen readers when page is reloaded.
    • See role="alert" and role="status" on MDN.

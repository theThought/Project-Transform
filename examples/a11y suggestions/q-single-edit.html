Accessibility considerations & other notes

  • Single input types - e.g. <input>, <textarea> - should use <label> for question text.
  • The input has an aria-describedby attribute to programmatically associate it with a unique id on the instructions. This enables screen readers to announce the instructions in addition to the form control's label.
  • We should not use placeholder text in input fields.
  • If status message is an error:
    • Place error message before question.
    • Shift keyboard focus to the 1st error message on the page so that it's the 1st thing announced to screen readers when page is reloaded.
    • See role="alert" and role="status" on MDN.

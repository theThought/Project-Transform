Accessibility considerations & other notes

  • The <input type="text"> should use <label> for question text.
    • Additionally, the question label has an id so that it can be programmatically associated with the special codes <fieldset> using aria-describedby.
  • The input and fieldset have aria-describedby attributes to programmatically associate them with a unique id on the instructions. This enables screen readers to announce the instructions in addition to the input's label and fieldset's legend.
  • We should not use placeholder text in input fields.
  • If status message is an error:
    • Place error message before question.
    • Shift keyboard focus to the 1st error message on the page so that it's the 1st thing announced to screen readers when page is reloaded.
    • See role="alert" and role="status" on MDN.

Accessibility considerations & other notes

  • Single input types - e.g. <input>, <textarea> - should use <label> for question text.
  • The input has an aria-describedby attribute to programmatically associate it with a unique id on the instructions. This enables screen readers to announce the instructions in addition to the input's label.
  • We should not use placeholder text in input fields.
  • Dealing with pre-label and post-label:
    • The pre-label might be something like "USD$". The post-label might be "per kg", etc.
    • In a scale question type, the labels might be "very disatisfied" and "very satisfied".
    • HTML source order positions the labels as shown in the demo, either side of the input.
    • If we use aria-describedby on the input to reference these labels, the reading order in a screen reader will announce the real <label> first, followed by the pre-/post- text, and would not make sense.
    • It might be better to not expose the pre-/post-labels at all in screen reader "focus" mode, and simply rely on the question label and/or additional instructions to provide sufficient context to screen readers.
  • If status message is an error:
    • Place error message before question.
    • Shift keyboard focus to the 1st error message on the page so that it's the 1st thing announced to screen readers when page is reloaded.
    • See role="alert" and role="status" on MDN.

# Templates

## Configuring Storybook controls to mimic back-end custom properties
- For example, see the [example question template story](/story/templates-1-example-question-template--example).
- The `<script>app.RegisterProperties(...)</script>` markup contains a template literal that matches the equivalent stringified JSON custom properties generated by the back-end, with properties/values mapped to the story's arguments.
- This allows Storybook controls to modify the custom properties, therefore updating the UI.

### Generate JSON output for ZeroHeight docs
- Uses the `parseCustomProperties()` function in `./Storybook/.storybook/preview-head.html` to sanitise and render custom properties JSON.
- This JSON can be copied by back-end users using the "generate & copy" button in the story.
- The HTML fragment for the button can be found inside `./Storybook/stories/_htmlFragments.js`.
